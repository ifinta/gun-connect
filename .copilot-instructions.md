## Copilot Project Context Prompt: gun-connect
Project Overview:
gun-connect is a decentralized "Service Discovery" PWA application built on Rust (Dioxus) and JavaScript (GunDB). Its primary goal is to use the Stellar Network Testnet as a "network layer" to share and synchronize GunDB relay addresses between users without a central server.
Architecture & Tech Stack:

* Frontend: Rust Dioxus (Wasm) for UI and core business logic.
* Database: GunDB (via JS interop) for local and P2P data management.
* Transport Layer: Stellar Network (Testnet) as a global, censorship-resistant relay directory.
* Data Protocol: Utilizing Stellar manageData operations (64 bytes per operation) to batch and broadcast relay URLs.
* Data Protocol: Use Horizon API to reach on other - similar - PWA App's some working GUN relay to communicate, sync GUNdb data between App's and users.

Core Logic to Implement:

   1. Stellar Discovery: The Rust code - in a need, at the start - must query the Stellar Horizon API, extract relay URLs from manageData fields, and push them into the GunDB peer list.
   2. Health Check: Rust logic to drop and find new relays in a need.
   3. Self-Healing Faucet: Invocation of the Stellar Friendbot (faucet) if the account balance is low or if the Testnet has been reset (404 error) if the user request it through a button click on the error modal dialog.

Coding Guidelines:

* Use existing patterns, libraries in this repo to develop the functions above.
* Ask, if something not clear or it exists better solution.
* Note: Encryption isn't a need for relay URLs as they are public by nature.
